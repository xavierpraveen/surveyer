Use CSS transition for fade/slide: `transition-all duration-300 ease-in-out` with max-h-0 overflow-hidden → max-h-96 and opacity-0 → opacity-100.

    Create `src/components/survey/SurveyWizard.tsx` as a client component. Props: survey (Survey), sections (SurveySection[]), questionsMap (Record<string, SurveyQuestion[]>), optionsMap (Record<string, QuestionOption[]>), initialDraft (ResponseDraft | null), initialSectionIndex (number).

    State: answers (Record<string, unknown>), currentSectionIndex (number), saveStatus ('idle'|'saving'|'saved'|'error'), showResumeBanner (boolean — true if initialDraft exists).

    On mount: if initialDraft, populate answers from draft.answers and set currentSectionIndex to initialDraft.last_section_index. Set showResumeBanner=true.

    Autosave: useEffect watching answers with 500ms debounce (use useRef for timeout). On change: set saveStatus='saving', call saveDraft({ survey_id, answers, last_section_index: currentSectionIndex }), then set saveStatus='saved', reset to 'idle' after 2000ms. On error: saveStatus='error'.

    Save status indicator: fixed bottom-right corner (Tailwind: fixed bottom-4 right-4 text-xs text-gray-500). Shows: "Saving..." when saving, "Saved ✓" when saved (fades out after 2s via opacity transition), "Save failed — retrying" when error (in red).

    Resume banner: dismissible banner at top of wizard. "Your progress has been saved. [Continue →]" — clicking Continue advances to last section. Dismiss button (×) hides the banner and sets showResumeBanner=false.

    Navigation: "Back" button (disabled on first section), "Next" button (or "Submit" on last section). On Next: calls saveDraft immediately (not debounced), advances currentSectionIndex.

    On Submit (Next on last section): validate all required questions in current section have answers. Call submitResponse({ survey_id, answers }). On success: router.push(`/surveys/${survey_id}/confirmation`). On error: show inline error.

    Each question in the section renders as `<ConditionalQuestion rule={q.conditional_rule} answers={answers}><QuestionRenderer ... /></ConditionalQuestion>`.

    Render SurveyProgressBar above the section content.

Computes participation_rate = count / eligible. Renders: full-page centered layout with survey title, "Thank you for participating!" message, participation rate displayed as "X% of employees have responded so far", note about when results will be shared ("Results will be shared after the survey closes."), "Back to Surveys" button. Client-side auto-redirect after 5000ms using useEffect redirect to /dashboard (add 'use client' for the redirect effect; or create a thin client wrapper around this RSC content).

    Note: The confirmation page also handles the "already submitted" read-only view. If accessed when submission exists, it just shows the same content without the auto-redirect (since user navigated deliberately).
