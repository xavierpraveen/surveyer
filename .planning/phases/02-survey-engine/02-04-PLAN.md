    Create `src/lib/actions/public-response.ts` with 'use server' directive. Import supabaseAdmin from '@/lib/supabase/admin' (import 'server-only' is already on admin.ts). Import cookies from 'next/headers'.

    Export `submitPublicResponse(surveyId: string, answers: Record<string, unknown>)`:
    1. Validate surveyId is a non-empty string
    2. Fetch survey from supabase to verify: public_link_enabled=true AND status='open'. If not: return { success: false, error: 'Survey is not available' }
    3. Check cookie `survey_submitted_${surveyId}` — if exists: return { success: false, error: 'already_submitted' }
    4. Insert into responses: { survey_id, user_id: null, is_anonymous: true, submitted_at: new Date().toISOString(), department: null, role: null, tenure_band: null, work_type: null }
    5. Insert response_answers rows for each answer (same structure as authenticated submission)
    6. Set cookie: await (await cookies()).set(`survey_submitted_${surveyId}`, '1', { maxAge: 60 * 60 * 24 * 365, path: '/', httpOnly: true, sameSite: 'lax' })
    7. Return { success: true }

    Note: No participation_token is created for public responses (no user_id to store).
    Note: Do NOT increment any public_response_count column — participation rate tracking for public surveys is deferred to Phase 3.
    Note: Use supabaseAdmin for all DB writes (no auth context available for public route).
