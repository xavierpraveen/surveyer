<objective>
Fix the BUG-03 column name mismatch in both authenticated and public survey submission paths. PostgREST silently drops INSERT columns it does not recognise тАФ the app has been writing `answer_text`, `answer_numeric`, `answer_options` while the DB schema defines `text_value`, `numeric_value`, `selected_options`. This leaves all three columns NULL on every submitted answer row, causing compute_derived_metrics to return zero rows and every downstream analytics dashboard to show empty data.

Purpose: Unblock the entire analytics pipeline (ANALYTICS-01тАУ10, DASH-01тАУ08) by ensuring answer values actually land in the database.
Output: Two modified Server Action files тАФ six renamed keys total (three per file), zero other changes.
</objective>
