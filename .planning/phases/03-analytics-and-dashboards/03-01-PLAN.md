Create the stored procedure that populates derived_metrics for a given survey. Write to: supabase/migrations/20260315000006_compute_derived_metrics.sql

The function signature:
```sql
CREATE OR REPLACE FUNCTION public.compute_derived_metrics(p_survey_id UUID)
RETURNS INT  -- returns count of rows inserted
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
```

The function must:
1. DELETE FROM derived_metrics WHERE survey_id = p_survey_id (idempotent — clears prior run)
2. INSERT aggregates for segment_type = 'overall' (no segment_value) — all respondents for that survey
3. INSERT aggregates for segment_type = 'department' — one row per dimension per unique department value in responses.department
4. INSERT aggregates for segment_type = 'role' — one row per dimension per unique role value in responses.role
5. INSERT aggregates for segment_type = 'tenure_band' — one row per dimension per unique tenure_band in responses.tenure_band
6. Return the count of rows inserted

For each segment, per dimension:
- JOIN response_answers ra ON ra.response_id = responses.id
- JOIN question_dimension_map qdm ON qdm.question_id = ra.question_id AND qdm.dimension_id = d.id
- JOIN questions q ON q.id = ra.question_id WHERE q.question_type IN ('likert_5', 'likert_10')
- Count distinct response_id values as respondent_count
- avg_score: ROUND(AVG(ra.answer_value::NUMERIC * qdm.weight) / AVG(qdm.weight), 2) — weight-adjusted mean
- favorable_pct: ROUND(100.0 * COUNT(*) FILTER (WHERE ra.answer_value::NUMERIC >= 4) / NULLIF(COUNT(*), 0), 1) — only for likert_5
  - For likert_10: favorable = answer_value >= 7, neutral = 5–6, unfavorable <= 4 (different scale)
  - Detect scale from question_type: if question_type = 'likert_10' use (>=7, 5-6, <=4); else use (>=4, =3, <=2)
- neutral_pct: ROUND(100.0 * COUNT(*) FILTER (WHERE ...) / NULLIF(COUNT(*), 0), 1)
- unfavorable_pct: ROUND(100.0 - favorable_pct - neutral_pct, 1)

GRANT EXECUTE ON FUNCTION public.compute_derived_metrics TO service_role;
-- Note: NOT granted to authenticated — this is admin-only via supabaseAdmin (service role)

The function should handle surveys with no responses gracefully (inserting zero rows and returning 0).

Apply the migration locally: `npx supabase db push` or note in action that executor should run `npx supabase db reset` to apply (use whichever matches the project's local Supabase workflow — check if supabase CLI is available with `which supabase`).
