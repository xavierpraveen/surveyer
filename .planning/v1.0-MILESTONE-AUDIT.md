## Critical Blockers

### 🔴 BUG-03: `response_answers` Column Name Mismatch

**Severity:** Critical — entire analytics pipeline silently broken
**Pattern:** Identical to BUG-01 (fixed in Phase 6) — PostgREST ignores unknown INSERT column names

**DB schema** (`supabase/migrations/20260315000001_schema.sql`):
```sql
response_answers: numeric_value, text_value, selected_options
```

**App code** (`src/lib/actions/response.ts` ~line 192–197, `src/lib/actions/public-response.ts` ~line 72–77):
```typescript
.insert({ answer_numeric: ..., answer_text: ..., answer_options: ... })
```

**Impact cascade:**
1. PostgREST receives `answer_numeric` (unknown column) → inserts NULL into `numeric_value`
2. `compute_derived_metrics` SQL: `WHERE ra.numeric_value IS NOT NULL` → zero rows pass
3. `derived_metrics` table → always empty after compute
4. `getLeadershipDashboardData` → `dimension_scores: []`, `participationRate: 0`
5. Leadership dashboard → "No completed survey results yet"
6. `/results` page → "Results will appear here after the first survey cycle closes"
7. Open-text `text_value` also NULL → `getTaggableAnswers` returns nothing → no tagging, no themes

**Fix:** In both files, rename insert payload keys:
- `answer_text` → `text_value`
- `answer_numeric` → `numeric_value`
- `answer_options` → `selected_options`

**Requirements affected:** RESPONSE-06, RESPONSE-07, RESPONSE-08, ANALYTICS-01 through ANALYTICS-10, DASH-01 through DASH-04, DASH-08, PRIVACY-03

### 🔴 BUG-04: Analytics Server Action Role Guards

**Severity:** Critical — leadership and manager dashboards inaccessible to the roles they serve

**File:** `src/lib/actions/analytics.ts`

| Function | Current guard | Correct guard |
|----------|--------------|---------------|
| `computeDerivedMetrics` (line ~44) | `['admin']` | `['admin', 'leadership', 'survey_analyst']` |
| `getLeadershipDashboardData` (line ~79) | `['admin']` | `['admin', 'leadership', 'survey_analyst', 'hr_admin']` |
| `getManagerDashboardData` (line ~505) | `['admin', 'employee']` | `['admin', 'manager', 'leadership', 'hr_admin', 'survey_analyst']` |

**Symptoms:**
- User with role `leadership` signs in → routed to `/admin` (Phase 6 fix) → opens leadership analytics → Server Action returns `Forbidden: admin role required`
- User with role `manager` opens manager dashboard → Server Action returns `Forbidden: insufficient role`

**Requirements affected:** DASH-01, DASH-04, DASH-05, DASH-06, DASH-07, ANALYTICS-01
