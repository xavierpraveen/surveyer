## Confirmed Bugs

### 🔴 BUG-01: `section_id` vs `survey_section_id` Column Mismatch

**Severity:** High — potential silent data bug
**Discovery:** Cross-referencing DB schema against app query code

DB schema (`supabase/migrations/20260315000001_schema.sql`, line 194):
```sql
survey_section_id  UUID  NOT NULL REFERENCES public.survey_sections(id) ON DELETE CASCADE
```

App queries (`src/lib/actions/survey.ts`, lines 161, 363, 440–445):
```typescript
.eq('section_id', questionFields.section_id)   // line 161
.in('section_id', sectionIds)                   // line 363
.insert({ ...questionFields, section_id: newSectionId })  // line 445
```

**Risk:** PostgREST silently ignores unknown filter columns, meaning `.in('section_id', sectionIds)` may return ALL questions rather than the section-filtered subset. Survey question rendering could show all questions from all sections, or none if PostgREST errors.

**UAT Status:** UAT passed in Phase 5 (survey wizard rendered correctly) — this may mean either PostgREST handles it, or the seed data only has one section. **Needs live DB verification.**

**Fix:** Rename app-side query column references from `section_id` to `survey_section_id` throughout `src/lib/actions/survey.ts` and `src/app/(admin)/admin/surveys/[id]/page.tsx`.

---

### 🟡 BUG-02: Incomplete Role Routing (AUTH-06 regression)

**Severity:** Medium — affects non-employee/admin users
**Discovery:** Code inspection of `src/lib/constants/roles.ts`

Current `ROLE_ROUTES`:
```typescript
export type AppRole = 'employee' | 'admin'
export const ROLE_ROUTES: Record<AppRole, string> = {
  employee: '/dashboard',
  admin: '/admin'
}
```

AUTH-06 requirement: *"Middleware routes users to role-appropriate areas (employee / manager / leadership / admin)"*

**Risk:** Any user with role `leadership`, `manager`, `hr_admin`, or `survey_analyst` in Supabase auth metadata would fail the AppRole type check and not be routed correctly. The middleware normalization (collapsing these to `admin`) masks this at runtime but removes the ability to give those roles distinct experiences.

**Context:** The v1 decision to consolidate all elevated roles to `admin` routing may be intentional scope reduction — but AUTH-06 claims routing exists for all roles. AUTH-07 still allows assigning these roles.

The v1.0 milestone is **functionally complete** for the core employee survey lifecycle, admin management, analytics, publication, and brand redesign. **98% of requirements are implemented** — the two genuinely missing features (ANALYTICS-11, DASH-07) are narrow scope items.
