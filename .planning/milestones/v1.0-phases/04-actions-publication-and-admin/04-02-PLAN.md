**src/lib/actions/publication.ts** — full implementation:

'use server' + 'server-only'. `const db = supabaseAdmin as any`.

1. `createPublicationSnapshot(surveyId: string)` — role guard (leadership, admin, survey_analyst):
   a. Fetch survey by ID. If survey.status !== 'closed' → return error 'Survey must be in closed state before publishing.'
   b. Check derived_metrics count: `SELECT count(*) FROM derived_metrics WHERE survey_id = ?`. If 0 → return error 'Run Compute Results before publishing.'
   c. Check existing snapshot: `SELECT id FROM publication_snapshots WHERE survey_id = ?`. If exists → return error 'Results already published for this cycle.'
   d. Gather snapshot data:
      - dimensionScores: call supabaseAdmin.rpc('get_dimension_scores_for_survey', { p_survey_id: surveyId, ... }) with segment_type='overall'
      - participationRate + totalResponses: fetch from v_participation_rates for surveyId (aggregate)
      - qualitativeThemes: SELECT from qualitative_themes WHERE survey_id
      - publicActions: SELECT from v_public_actions WHERE survey_id OR survey_id IS NULL
   e. Build SnapshotData object (schemaVersion: 1, all gathered fields)
   f. INSERT into publication_snapshots: { survey_id: surveyId, snapshot_data: snapshotData, published_by: user.id }
   g. Return `{ success: true, data: { snapshotId: inserted.id } }`

2. `getPublicationSnapshot(surveyId: string)` — SELECT from publication_snapshots WHERE survey_id. Map snake_case to camelCase (snapshotData = snapshot_data). Return `{ success: true; data: PublicationSnapshot | null }`.

3. `getPublishedCycles()` — SELECT publication_snapshots JOIN surveys for title and closed_at. Order by created_at DESC. Return `{ success: true; data: Array<{ surveyId, surveyTitle, closedAt, snapshotId }> }`.

**src/lib/actions/settings.ts** — full implementation:

'use server' + 'server-only'. `const db = supabaseAdmin as any`.

1. `getAppSettings()` — SELECT from app_settings WHERE key IN ['privacy_threshold_numeric', 'privacy_threshold_text', 'allowed_email_domain']. Map rows to AppSettings object with defaults (5, 10, ''). No role guard needed (read-only settings needed by all admin pages).

2. `updateAppSettings(key: string, value: number | string)` — role guard (admin, hr_admin). UPSERT into app_settings. Return `{ success: boolean }`.

3. `importEmployees(rows: EmployeeImportRow[])` — role guard (admin, hr_admin). For each valid row:
   a. Try `supabaseAdmin.auth.admin.createUser({ email: row.email, email_confirm: true, user_metadata: { name: row.name } })`.
   b. If error code 422 (user exists) → increment skipped, continue.
   c. If success → upsert into profiles: { id: createdUser.id, email, department lookup by name, role, tenure_band }.
   d. Return `{ success: true; data: ImportResult }` with final counts.

4. `archiveSurvey(surveyId: string)` — role guard (admin). Fetch survey. If status !== 'closed' → return error 'Only closed surveys can be archived.' UPDATE surveys SET archived = true WHERE id. Return `{ success: boolean }`.

5. `getParticipationForOpenSurvey()` — find the currently open survey (status = 'open'). If none → return `{ success: true; data: [] }`. Query v_participation_rates for that surveyId. Map to ParticipationRow[]. Return `{ success: true; data: ParticipationRow[] }`.
