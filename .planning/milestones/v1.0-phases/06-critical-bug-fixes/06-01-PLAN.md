<objective>
Fix BUG-01: the DB column on the `questions` table is `survey_section_id` but every app query, type, validation, and component uses `section_id`. PostgREST silently ignores unknown filter columns, so `.in('section_id', sectionIds)` currently returns ALL questions regardless of section — a silent data correctness bug that only manifests when a survey has more than one section.

Purpose: Multi-section surveys must filter questions by the correct column so each section shows only its own questions.
Output: Seven files updated; zero remaining `section_id` references in question query/type/validation context.
</objective>
